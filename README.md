# 📚 The Ledger - Library Management System (Sopranos Edition)

**The Ledger** — це повнофункціональний Full-stack застосунок для управління бібліотечним фондом, розроблений у межах навчальної практики (День 4). Проєкт реалізовано з акцентом на сувору архітектуру, валідацію даних та унікальний візуальний стиль у дусі класичної кримінальної драми "The Sopranos".

---

## 🎭 Концепція
Замість нудного списку книг, ви працюєте з **"The Ledger"** (Обліковою книгою). Кожна книга — це "Справа", кожен автор — "Associate", а жанр — "Territory". Система побудована так, щоб ви відчували вагу кожного запису.

---

## 🛠 Технологічний стек

### Backend (Серверна частина)
* **Framework:** Nest.js (TypeScript) — модульна архітектура.
* **Validation:** `class-validator` & `class-transformer` — сувора перевірка вхідних даних через DTO.
* **Storage:** In-memory сховище (масиви) з логікою CRUD.
* **Pipes:** Глобальний `ValidationPipe` для обробки помилок запитів.

### Frontend (Клієнтська частина)
* **Library:** React + TypeScript (збірка через Vite).
* **UI Framework:** Material UI (MUI) — кастомна темна тема.
* **Routing:** React Router DOM v6 (3 активні маршрути).
* **Forms:** React Hook Form — ефективне управління станом форм.
* **API Client:** Axios — обробка асинхронних запитів.

---

## 🚀 Функціональні можливості (CRUD)

Застосунок повністю відповідає технічним вимогам практики:
1.  **Сутності:** Реалізовано роботу з двома сутностями: `Books` (Книги) та `Authors` (Автори).
2.  **Перегляд (Read):** Динамічний список усіх справ із відображенням деталей.
3.  **Створення (Create):** Форма додавання нових записів із валідацією типів.
4.  **Редагування (Update):** Окрема сторінка для оновлення існуючих даних.
5.  **Видалення (Delete):** Безпечне видалення з системним підтвердженням ("They pull me back in!").
6.  **UX/UI:** Реалізовано стани завантаження (Loaders) та обробку помилок.

## 📡 API Ендпоінти

| Метод  | Шлях           | Опис                                   |
| :----- | :------------- | :------------------------------------- |
| GET    | `/books`       | Отримати всі записи з Ledger           |
| GET    | `/books/:id`   | Деталі конкретної справи               |
| POST   | `/books`       | Внести новий запис (вимагає DTO)       |
| PUT    | `/books/:id`   | Оновити дані існуючого запису          |
| DELETE | `/books/:id`   | Видалити справу з обліку               |
